# Product

## Register

brand

## Users

Deux audiences simultanées : le Chef de Projet ou UX Designer non-technique (qui veut créer des tours guidés sans ouvrir un ticket dev), et le développeur qui évalue l'outil (cherche JSON versionnable, zéro infra, ownership du code). Le CDP est convaincu par la valeur autonomie ; le dev est convaincu par la sobriété technique et l'absence de vendor lock-in.

## Product Purpose

Callis est un bookmarklet universel qui permet de créer visuellement des tours guidés et tooltips contextuelles sur n'importe quel site, sans toucher au code. Le résultat est un `callis.json` neutre compatible driver.js, Shepherd.js, Intro.js. Le `CALLIS_TASKS.md` généré est un ticket prêt à exécuter par un dev pour stabiliser les sélecteurs. Callis rend le CDP autonome à 80% sans infrastructure, sans SaaS, sans abonnement.

## Brand Personality

Simple, direct, dev-friendly. Un outil qui fait ce qu'il dit, sans fioritures. Confiant sans être arrogant. La qualité est dans la sobriété de l'exécution, pas dans le marketing.

## Anti-references

- **Notion / Linear** : minimalisme ultra-épuré, product-led trop froid. Callis a plus de chaleur et de caractère typographique que Notion.
- **Appcues / Pendo** : SaaS flashy, screenshots héroïques, pricing agressif. Callis est l'opposé : open, sobre, sans pression commerciale.
- **Intercom** : bleu corporate, institutionnel, "support tool". Callis parle aux makers, pas aux équipes support.

## Design Principles

1. **Montre, ne vante pas.** Le démo inline suffit. Pas de "puissant", "révolutionnaire", "game-changer".
2. **L'outil parle pour lui-même.** Le JSON versionnable, le zéro infra, le bookmarklet : ce sont des faits, pas des promesses.
3. **Dev-confidence.** Le dev doit pouvoir faire confiance au code avant de faire confiance au produit. Snippets, format JSON visible, MIT license en évidence.
4. **CDP-clarity.** Chaque phrase doit être comprise par quelqu'un qui n'a jamais ouvert un terminal. Pas de jargon non défini.
5. **Pas de SaaS anxiety.** Zéro CTA agressif, zéro pricing fear, zéro "essayez gratuitement pendant 14 jours". C'est open source, point.

## Accessibility & Inclusion

WCAG AA minimum. Contraste texte sur fond respecté. Animations réduites si `prefers-reduced-motion`. Pas de couleur comme seul vecteur d'information.
